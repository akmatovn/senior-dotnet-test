О компании

YouDo — платформа, которая помогает людям и бизнесу быстро находить исполнителей для бытовых и рабочих задач. Аудитория — миллионы пользователей.

Мы развиваем core-платформу: переходим от монолита к распределённой архитектуре (20+ сервисов и приложений), повышаем надёжность и производительность.

Мы активно используем AI-инструменты в разработке (Cursor/Codex/Claude и аналоги) как часть ежедневного процесса: для ускорения реализации, рефакторинга, генерации тестов и анализа инцидентов.

Вакансия

Ищем сильного Senior Individual Contributor: без people-management, но с большим техническим влиянием и ownership.

Чем предстоит заниматься

  • Владеть сервисами/доменом end-to-end: проектирование, разработка, поддержка в проде
  • Декомпозировать монолит: выделять сервисы, проектировать контракты, планировать миграции
  • Проектировать интеграции: HTTP/RabbitMQ/Kafka, ретраи, дедупликация, идемпотентность
  • Работать с данными: PostgreSQL/MS SQL — индексы, планы запросов, блокировки, транзакции
  • Улучшать производительность и стабильность: профилирование, GC/аллокации, p95/p99

Требования

  • Обязательно: уверенный практический опыт работы с AI-инструментами (Cursor/Codex/Claude)
  • Senior-уровень в .NET (5+ лет): C#, ASP.NET Core, async/await, concurrency
  • Опыт проектирования и поддержки production-систем
  • Сильная база по реляционным БД: оптимизация запросов, индексы, транзакции
  • Опыт с брокерами сообщений (RabbitMQ/Kafka)

Технологии

.NET 8 C# ASP.NET Core PostgreSQL MS SQL Redis Elasticsearch RabbitMQ Kafka GitLab CI/CD

Условия

  • Оформление по ТК РФ, «белая» зарплата
  • Офисы Москва/Казань или полностью удалённо (любая страна)
  • Сложные инженерные задачи: масштаб, перформанс, миграция к микросервисам
  • Бюджет на обучение и AI-инструменты

Тестовое задание

Мы заботимся о пользователях и хотим максимально быстро помочь им найти ответы на вопросы. У нас есть база знаний с 66 статьями FAQ. Нужен бот, который поможет пользователям находить нужную информацию.

Задача

Создать Telegram-бота на .NET, который отвечает на вопросы пользователей по базе знаний YouDo.

Что должен делать бот

  1. Принимать текстовый вопрос от пользователя
  2. Искать релевантные статьи в базе знаний
  3. Возвращать ответ на основе найденных статей

Технические требования

  • Язык: C#, .NET 8
  • Telegram Bot API: polling или webhook — на ваш выбор
  • База знаний: файл knowledge_base.json в репозитории

Критерии оценки

  • Работоспособность — бот запускается и отвечает на вопросы
  • Качество кода — читаемость, структура, SOLID
  • Архитектура — разделение ответственности, расширяемость
  • Использование AI-инструментов — как использовались в разработке (описать в README)
  • README — инструкция по запуску, описание решения

Как сдать

  1. Сделайте fork этого репозитория
  2. Реализуйте бота
  3. Добавьте README с инструкцией по запуску и описанием решения
